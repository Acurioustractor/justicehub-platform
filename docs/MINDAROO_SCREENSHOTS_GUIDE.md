## Screenshot Requirements & Standards

### Technical Specifications

**For Web Display:**
- Format: PNG with transparency where applicable
- Resolution: 1920×1080px (standard screens)
- File size: < 500KB (optimize with tinypng.com)
- Naming: lowercase, kebab-case, descriptive

**For Presentations:**
- Format: PNG, high-resolution
- Resolution: 2400×1600px minimum (300dpi)
- File size: < 2MB per image
- Color profile: sRGB

### Capture Guidelines

**Platform Screenshots:**
1. Use incognito/private browsing (clean state)
2. Standard viewport: 1920×1080px
3. Remove any test/dummy data
4. Ensure professional appearance
5. Crop to relevant area (remove unnecessary UI)

**Data Visualizations:**
1. Ensure all text is legible
2. Use Mindaroo brand colors where possible
3. Include data sources/citations
4. Remove decorative elements

**Annotations:**
- Use simple red circles/arrows if highlighting features
- Keep annotations minimal and professional
- Use Arial or Public Sans for any text overlays

## Integration Checklist

### For each screenshot, create:

1. **High-quality source file** (PNG, properly named)
2. **Optimized web version** (compressed for fast loading)
3. **Alt text description** (for accessibility)
4. **Caption/context** (if needed in documentation)
